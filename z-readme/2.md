1. <i> 标签的默认行为
   <i> 在 HTML 里原本是 斜体（italic）的行内元素。

很多浏览器对 没有 class 的 <i> 会当作“无内容的 inline 元素”处理，特别是里面只有 <svg> 的时候，宽高计算可能异常。
简单说，如果 <i> 没有 class，也没有内容的文本节点，浏览器可能会让它的尺寸塌掉。

2. 加 class 后的变化
   任何 class（即使 CSS 不存在）都会让浏览器把它当作 可能需要样式的元素 去渲染。
   渲染引擎就不会再把 <i> 的宽高优化掉，所以 <svg> 可以正常显示，并撑开父元素。
   换句话说，这不是 CSS 生效的原因，而是 渲染引擎因为 class 而不再“优化掉”这个元素。

3. 为什么这种现象在 <i>、<span>、空 inline 元素上更常见
   <i>、<span> 默认是行内元素。
   如果没有文字或明确尺寸，它们可能被浏览器压缩为 0×0。
   加了 class 或者设置 display: inline-block / block 就能保证宽高计算正确。

1. Input 组件是通用的
   Element 的 Input 既可以在表单里用，也可以单独用
   Input 不应该强绑定某个字段或规则，否则就变得“不可复用”

2. 表单校验需要统一管理
   FormItem 可以收集错误信息
   Form 可以统一触发 validate()
   如果每个 Input 自己校验，Form 无法“一次性校验所有字段”

3. Input 只负责“值变化 / blur / focus”
   Input 提供事件（input/blur）
   FormItem 根据事件触发校验
   Input 完全不知道校验规则是什么
   🔑 结论：校验逻辑放在 FormItem，Input 只负责传值和触发事件

ElInput → 只负责事件
ElFormItem → 规则解析 + 单字段校验
ElForm → 批量触发 + 汇总结果
